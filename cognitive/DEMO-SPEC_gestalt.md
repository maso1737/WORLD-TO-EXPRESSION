# PART 1 — 仕様書テンプレート

## 0. 通底テーマ（各シリーズ冒頭に1段落・固定する）

> このシリーズは「正しい知識」を配るものではなく、**自分の脳がどう騙され・どう勝手に補完するかを手で確かめる**ためのもの。
> スライダーを動かして効果が **立ち上がる瞬間** と **消える瞬間** の両方を自分で跨ぐ。
> ゴールは "I get it by looking" ではなく **"I can reproduce it by touching."**。
> **"VJ感" は歓迎する。** 教材然とした硬さは禁物——勉強っぽく見えた時点で二度と開かれない。
> 発光・残像・粒子・リズム等の感覚的な気持ちよさを積極的に使い、**「また触りたい / また見たい」** を最優先する。
> ただし飾りは手触りに従属させる: 演出は必ず「効果が立ち上がる/消える瞬間」を**増幅する方向**にだけ効かせ、
> その瞬間を覆い隠してはならない。VJ的な気持ちよさ ＝ 餌、"I can reproduce it by touching" ＝ 釣り針。

## 0.5 認識エンジン4層（シリーズ最上位の組織図）

脳が世界を読むときに走らせる工程の順番＝シリーズの背骨。各原理は必ずどれか1層に属す。
階層は `シリーズ → Act(Layer) → App(原理)`。

| L | 動詞 | 問い | 性質 | テーマ | 軸 |
|---|---|---|---|---|---|
| 1 | **GROUP** | 何が仲間か | 自動（engine→あなた） | ゲシュタルト | 空間 |
| 2 | **FOCUS** | 何が主役か | 自動 | 視覚重力 | 空間 |
| 3 | **INTERPRET** | 同じ入力に何通りの解釈が立つか（解釈は**構築**され、誤り・視点依存になる） | 自動 | 図地反転・アナモルフォーシス・錯視 | 空間 |
| 4 | **EDIT** | 何を捨てれば engine を**操れる**か | **作為（あなた→engine）** | 間・抑制・余白 | 空間⇔時間の蝶番 |

**4層は同じ性質ではない。** L1〜3は脳が勝手にやってくる（自動）。L4 EDIT だけは、あなたが
engine に対してやる作為。この段差がシリーズの山場で、**読者が被験者から操縦者に変わる**地点。
KINETIC STAGE の「気づく→体感→**再現**」の "再現" が L4 に対応する。よって EDIT は横並びの1枚ではなく、
**L1〜3 を統合して使うキャップストーン**として置く（余白/間は閉合や焦点を"引き算で"発動させる＝下層の応用）。

**ラダーではなくループ。** 教える順は L1→L4 の積み上げだが、実態は L3 の解釈(プライア)が
L1 群化・L2 焦点に**降りてくる**（見えた瞬間に群化が変わる／アナモルフォーシスが主役を書き換える）。
→ ハブ(index)は階段ではなく**環**で描く。アナモルフォーシス＝下層を書き換える実演そのもの。

**この4層は"空間"エンジン。時間エンジンは別軸。**
- **時間エンジン**: アニメ12原則・音楽理論・リズム（別シーズン／姉妹 engine）。
- **間(Ma)** は空間の隙間でalso時間の休符＝**2エンジンの蝶番**。だから L4 に置きつつ時間軸にも足が掛かる。

**宙ぶらりんだったテーマの居場所:**
- **黄金比/フィボナッチ** — 知覚でなく「骨格・比率」。`Layer0 STRUCTURE（何が骨格か）`を L1 手前に足すか、
  構築トラック(KINETIC GEOMETRY 数学側)に戻す。元の平坦な並びで②に押し込んでた違和感はこれで解消。
- **映画/演出(E案)** — 視線誘導(FOCUS)＋間/情報量(EDIT)…と4層を**全部使う統合作品**＝最終章。レイヤーの1枚ではない。
- **音楽(D案)** — 視覚4層に乗らない＝時間エンジン側。Web Audio 共有ランタイムで別シーズン。

**この組織図の実利:**
- ハブ＝「4層の環マップ」。各アプリ冒頭に `これは L2 FOCUS / L1 から群化を借りている` の相互参照タグ → ループを体感させる。
- **どの層にも入らないデモは企画ごと疑う**（スコープのフィルタ）。

## 1. 仕様の「肝」— なぜこの型なのか

このシリーズのボトルネックは **事実の正確さではなく、何を触らせて何が起きるかの設計**。
Gemini の正確な解説は素材として要るが、それは「どのスライダーをどの値にすると脳のバグが見える/消えるか」を教えてくれない。
だから各原理で必ず次の3点を先に決める。事実(出典)は最後に脚注で足りる。

1. **触る対象**（主操作スライダー/トグル）
2. **反転点・閾値**（効果が最大化 or 切り替わる値）
3. **打ち消し(off)状態**（効果が消える値＝境界条件）← **これが教材の核心**

「off状態」を全原理に必ず持たせる。効果がある状態しか無いと "おもちゃ"、
**off に跨げて初めて "教材"**（=どこを超えると騙されるのかが手で分かる）。
書く側にとっても、定義を読むより境界条件を言語化するほうが深く理解できる＝**準備がそのまま勉強になる**。

## 2. 原理ごとの記入ブロック（この型を複製して埋める）

```
### NN. 英名 (EN) / 和名
- 一言で        : その現象を1文で（= ステージで起きること）
- なぜそう見える : 知覚/神経の機構を2〜3行（← Gemini リサーチの着地点。出典を脚注で）
- ステージ      : canvas に何が出るか（要素・初期配置）
- 触る対象      : 主スライダー/トグル（変数名候補つき）
- 反転点 / 閾値 : 動かすと効果が最大化 or 90°回転 or 切替わる値
- 打ち消し(off) : 効果が消える値（必須・境界条件）
- proof/reveal  : 知覚を客観で重ねて検証するトグル（定規・群の着色・補完輪郭のゴースト 等）

- 実装メモ      : 2D canvas / SVG / CSS / 要アニメ(rAF) / 要Web Audio など
- COPY/PASTE    : シリアライズ対象パラメータ（KINETIC STAGE 規約・末尾必須）
- 出典          : (Gemini doc 〜) 章/論文へのポインタ
```

## 3. 共通インタラクション・パターン目録（全テーマで使い回す5型）

既存スキル群（camera-rig / color-palette / composer-timeline …）の **教育レイヤー版**。
新テーマでもこの5型のどれかに必ず当てはまるので、毎回ゼロから考えない。

| # | パターン | 何をするか | 使う原理の例 |
|---|---|---|---|
| 1 | **閾値スライダー** | 1本で効果の立上り/消失の境界を跨ぐ | 近接(間隔比)・閉合(ギャップ)・共通運命(コヒーレンス)・対称(対称度) |
| 2 | **対抗スライダー** | 2手がかりを綱引きさせ優先順位を見る | 類同 vs 近接・図地の複数手がかり |
| 3 | **真実オーバーレイ** | 知覚した群/輪郭/等価性を客観で重ねて検証 | 全原理。錯視シリーズで「定規版」に発展 |
| 4 | **A/Bトグル・多義切替** | 効果のon/offを直接比較、多義図形の見えを記録 | 図地・閉合(カニッツァ) |
| 5 | **一斉アニメ** | 要素を揃えて動かす/バラす（rAF） | 共通運命。12原則 timing と共有 |

> **proof overlay** は錯視シリーズで「定規・同心円で等価を証明する部品」に育つ。
> ミュラー・リヤー/ポンゾ/エビングハウスで使い回せるので、ここで作る描き方を共通部品化しておく。

# PART 2 — ゲシュタルト原則 初版（A案・7原理）
> **Act / Layer 1 — GROUP（何が仲間か）** ｜ 自動・空間軸 ｜ engine→あなた

> 進行順の推奨1番手。理由: 最もブラウザネイティブ、デモが点と線で単純、各原理がスライダー1本で成立、
> 錯視・視覚重力の土台になる。新シリーズのパイプラインを一番軽いテーマで通す意味でも◎。
> L1 は最下層＝**全ての入口**。L2 FOCUS・L3 INTERPRET はここで作られた「群」を前提に動く。
> 配色: gold=ラベル/数値, rouge=主役要素, ice=強調/真実オーバーレイ, lavender=JP補助（KINETIC STAGE 標準）。

## 概観

| NN | EN | JP | 触る（一言） | off（一言） | パターン |
|---|---|---|---|---|---|
| 01 | Proximity | 近接 | 縦横の間隔比 | 比1:1で群が消える | 閾値 |
| 02 | Similarity | 類同 | 色差/形差 ⇔ 間隔 | 差0で近接だけ残る | 対抗 |
| 03 | Continuity | 連続 | 経路の角度/曲率 | ランダム化で線が辿れない | 閾値+真実 |
| 04 | Closure | 閉合 | 輪郭のギャップ | ギャップ最大で図が消える | 閾値+真実 |
| 05 | Figure/Ground | 図と地 | 図地を決める手がかり | 手がかりMAXで反転が止まる | A/B切替 |
| 06 | Common Fate | 共通運命 | 運動コヒーレンス | コヒーレンス0でノイズ | 一斉アニメ |
| 07 | Symmetry | 対称性 | 左右の対称度 | 対称度0でまとまり消失 | 閾値+真実 |

### 01. Proximity / 近接
- 一言で        : 距離が近い要素を1グループと見る。**色や形より間隔が優先**されることが多い。
- なぜそう見える : 視覚系は空間的近さをグルーピングの一次手がかりにする。要素の物理的特徴より「どれだけ近いか」を先に使う。¹
- ステージ      : ドット格子（例 6×4）。色も形も全部同じ。
- 触る対象      : `colGap`（列間隔）/ `rowGap`（行間隔）の2スライダー。
- 反転点 / 閾値 : 縦横の間隔比が **1:1 を跨ぐ** と、知覚される群の向きが90°回転（行のまとまり⇔列のまとまり）。
- 打ち消し(off) : 縦横を完全に等間隔（比1:1）→ 特定の群化が消えて「均一な面」に見える（どちらの群でもない）。
- proof/reveal  : 知覚される群の境界を ice で薄く塗るトグル。自分の見えと一致するか確認。
- 再現できたか  : 「この間隔比ならユーザーは行で読む」と予測して配置できる。**UI余白設計に直結**。
- 実装メモ      : CSS grid or canvas dots。2スライダーのみ。アニメ不要。
- COPY/PASTE    : `{colGap, rowGap, cols, rows, showGroups}`
- 出典          : (Gemini doc — Proximity / Wertheimer 1923)

### 02. Similarity / 類同
- 一言で        : 似た見た目（色・形・サイズ・向き）の要素を1グループと見る。**近接と競合**する。
- なぜそう見える : 特徴の類似がグルーピング手がかりになる。近接とわざと対立させると、両者の優先順位が体感できる。¹
- ステージ      : ドット格子。一部の行(or列)だけ色/形を変える。間隔は近接と逆向きに設定。
- 触る対象      : `featDelta`（色差or形差＝類同の強さ）/ `gapBias`（近接の対抗）。← **対抗スライダー**。
- 反転点 / 閾値 : 類同を強めると、近接由来の群を **上書き** して色/形で群が決まる点。
- 打ち消し(off) : `featDelta=0`（全要素を同一に）→ 類同の群が消え、近接だけが残る。
- proof/reveal  : 「近接の群」と「類同の群」を別色境界で同時表示。今どちらが勝っているか可視化。
- 再現できたか  : 近接と類同を競合させたとき、**どちらでグルーピングを誘導するか**を設計できる。
- 実装メモ      : canvas。色 hue 差 or 形（●/■）差。アニメ不要。
- COPY/PASTE    : `{featDelta, featType, gapBias, cols, rows}`
- 出典          : (Gemini doc — Similarity)

### 03. Continuity / 連続（good continuation）
- 一言で        : 滑らかな線に沿って並ぶ要素を「連続した一本」と見る。交差点では **折れる方より直進/滑らかな方** を辿る。
- なぜそう見える : 視覚系は急な方向転換より滑らかな経路を好む。交差を「2本の良い連続」に分解する。²
- ステージ      : 2本の経路（点列）が中央で交差。交点で「滑らかに繋がるペア」と「折れるペア」が両立。
- 触る対象      : `curveB`（一方の経路の曲率/角度）。滑らかさを崩していく。
- 反転点 / 閾値 : 滑らかに繋がる側の角度が急になりすぎると、知覚される「1本」の繋がりが切替/崩壊する閾値。
- 打ち消し(off) : 経路をランダム化（`jitter=max`）→ 点はバラバラに見え、線として辿れない。
- proof/reveal  : 知覚される連続線を ice 1色でハイライトするトグル。
- 再現できたか  : 視線を線に沿って誘導する配置ができる（**視線誘導=視覚重力シリーズの前段**）。
- 実装メモ      : canvas。2本の Bézier 上に点を並べ、片方の角度を可変。アニメ不要。
- COPY/PASTE    : `{curveB, jitter, dotCount, highlightPath}`
- 出典          : (Gemini doc — Good Continuation)

### 04. Closure / 閉合
- 一言で        : 途切れた輪郭でも、脳が欠けを補って **閉じた図形** として見る。
- なぜそう見える : 視覚系は不完全な形を既知の完全形に補完する。極端には実在しない輪郭（カニッツァ）まで作る。³
- ステージ      : 円 or 三角の輪郭を破片（弧/線分）で描く＋ギャップ。切替で「パックマン3つ=カニッツァ三角」モードも。
- 触る対象      : `gap`（破片の間隔）スライダー。
- 反転点 / 閾値 : ギャップが小さいうちは「閉じた円」、ある閾値を超えると「バラバラの弧の集まり」に崩れる。**その閾値が肝**。
- 打ち消し(off) : `gap=max`（破片を散らす）→ 図形が消えてただの線分群。閉合が起きない境界。
- proof/reveal  : 補完される輪郭（脳が描く線）を薄いゴーストで重ねるトグル。カニッツァ時は錯覚輪郭を表示。
- 再現できたか  : **どこまで省略してもロゴ/アイコンが読めるか**（最小情報量）を判断できる。
- 実装メモ      : SVG or canvas。ギャップ可変＋カニッツァ・トグル。アニメ不要。← **錯視シリーズ(錯覚輪郭)への橋**。
- COPY/PASTE    : `{shape, gap, mode(outline/kanizsa), showGhost}`
- 出典          : (Gemini doc — Closure / Kanizsa 1955)

### 05. Figure/Ground / 図と地
- 一言で        : 同じ輪郭が「図(前景)」にも「地(背景)」にも見え、両立しない多義図形（ルビンの壺）。
- なぜそう見える : 視覚系は領域のどちらかを図に割り当てる。手がかり（囲み・サイズ・凸性・対称・下方）で図が決まる。⁴
- ステージ      : ルビンの壺型 or 対称な白黒2領域。
- 触る対象      : 図地を決める手がかりスライダー（`areaBias` 面積比 / `convexity` 凸性 / `contrast` のいずれか1〜2）。
- 反転点 / 閾値 : 手がかりが中立(50/50)付近だと図地が **自発的に反転**（多義安定）。振ると一方に固定。
- 打ち消し(off) : 手がかりMAX → 反転が起きず安定。曖昧さが消える境界。
- proof/reveal  : 「今あなたが図として見ているのは?」A/Bトグルで自分の見えを記録 → 手がかり値との対応を表示。
- 再現できたか  : 主役/背景を意図的に固定する/わざと曖昧にする制御ができる（**視覚重力 主役-脇役 への布石**）。
- 実装メモ      : SVG/canvas。手がかりスライダー＋A/B記録。アニメ不要。
- COPY/PASTE    : `{areaBias, convexity, contrast, seenAs}`
- 出典          : (Gemini doc — Figure/Ground / Rubin 1915)

### 06. Common Fate / 共通運命
- 一言で        : 同じ方向に同じタイミングで動く要素を1グループと見る。静止時は無関係でも、**一緒に動けば群になる**。
- なぜそう見える : 共通の運動ベクトルは最強級のグルーピング手がかり。動きは近接・類同を上書きしうる。⁵
- ステージ      : ランダムドット雲。静止時は均一でノイズ。
- 触る対象      : `coherence`（一致して動くドットの割合/方向の揃い）/ `speed`。← **一斉アニメ**。
- 反転点 / 閾値 : コヒーレンスがある閾値を超えると、突然「面/形」が浮かび上がる（運動コヒーレンス閾値）。
- 打ち消し(off) : `coherence=0`（全ドットをバラバラに）or 静止 → 群が消えてノイズに戻る。
